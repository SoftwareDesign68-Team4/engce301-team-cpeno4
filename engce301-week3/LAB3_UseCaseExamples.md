## UC-01: View Event List

**Primary Actor:** Student  
**Goal:** ดูรายการประวัติการเข้าร่วมกิจกรรมที่เคยเข้าร่วม

### Preconditions
- นักศึกษาเข้าหน้าล็อคอินอีเมลมอเข้าหน้า index ได้
- มีกิจกรรมที่เคยเข้าร่วมอย่างน้อย 1 กิจกรรม

### Postconditions
- นักศึกษาเห็นกิจกรรมที่เคยเข้าร่วมและชั่วโมงที่ได้รับ

### Main Flow
1. นักศึกษาล็อกอิน
2. กดปุ่มแสดง History
3. ระบบแสดง ตารางกิจกรรมที่เข้าร่วมและชั่วโมง
4. ตารางจะมีชื่อกิจกรรมที่เคยเข้าร่วมและชั่วโมงที่ได้รับ
5. ท้ายตารางมีชั่วโมงที่สะสมมาทั้งหมด

### Alternative Flow
- **AF-1 – ไม่มี Event ในระบบ**
  1. ผู้ใช้ยังไม่เคยเข้าร่วมกิจกรรมใดเลย
  2. ระบบแสดงข้อความ “คุณยัง ไม่เคยเข้าร่วมกิจกรรม”
  3. Use Case จบลง

  ## UC-06: Create New Event

**Primary Actor:** university officials  
**Goal:** อยากสร้างกิจกรรมการเรียนรู้ความคิดสร้างสรรค์และสิ่งต่างๆ และจำนวนนักเรียนที่เข้าร่วมกิจกรรม

### Preconditions
- นักศึกษาหรือบุคลากร เข้าหน้าล็อคอินอีเมลมอเข้าหน้า index ได้

### Postconditions
- สามารถกรอกแบบฟอร์มเสนอยื่นขอจัดกิจกรรมได้

### Main Flow
1. นักศึกษาหรือบุคลากรล็อกอิน
2. กดปุ่ม Access Event 
3. ระบบแสดงแบบฟอร์มให้กรอก
4. แบบฟอร์มจะถามรายละเอียดกิจกรรมที่เสนอมา
5. เมื่อเสร็จสิ้น ระบบจะแสดงข้อความ “เสนอกิจกรรมสำเร็จ”
6. รอผู้มีสิทธิอนุมัติจัดกิจกรรมมาตรวจและอนุมัติพร้อมให้feedback

### Alternative Flow
- **AF-1 – ไม่มี Event ในระบบ**
  1. ผู้ใช้ไม่ใช่คนในมหาลัย
  2. ผู้ใช้ไม่มีอีเมลของมหาลัย
  3. Use Case จบลง

# เพิ่มขึ้นมาใหม่
## UC-09: Event Submission

**Primary Actor:** Activities Teacher/Activity Chairman  
**Goal:** ผู้มีสิทธิดูกิจกรรมที่ถูกเสนอเข้ามาและ อนุมัติหรือไม่อนุมัติกิจกรรมนั้นได้

### Preconditions
- นักศึกษาหรือบุคลากร เข้าหน้าล็อคอินอีเมลมอเข้าหน้า index ได้

### Postconditions
- สามารถดูรายละเอียดกิจกรรมที่เสนอเข้ามา

### Main Flow
1. นักศึกษาหรือบุคลากรล็อกอิน
2. กดปุ่ม Event Submission
3. ระบบแสดงรายละเอียดกิจกรรมนั้น
4. ท้ายแบบฟอร์มจะมีปุ่ม pass / not pass
5. เมื่อเสร็จสิ้น สถานะกิจกรรมนั้นจะบอกในหน้า index
6. รอผู้ยื่นเสนอมาแก้ไขหรือยกเลิกการเสนอ

### Alternative Flow
- **AF-1 – ไม่มี Event ในระบบ**
  1. ผู้ใช้ไม่ใช่คนในมหาลัย
  2. ผู้ใช้ไม่มีอีเมลของมหาลัย
  3. ผู้ใช้ไม่มีสิทธิในการอนุมัติกิจกรรม
  4. Use Case จบลง
